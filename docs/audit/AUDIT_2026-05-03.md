## P0 — Ship-blockers (14)

### CLI

- **CLI-1** — `read_file` has no path validation. LLM can `read_file("~/.ssh/id_rsa")` or `~/.agiworkforce/auth.json`. `apps/cli/src/tools.rs:209-338`
- **CLI-2** — Shell-style injection via `search_files` and `grep_files` pattern args. `apps/cli/src/tools.rs:643-650`
- **CLI-3** — Daemon log files write `prompt` + `response` verbatim, no `redact_secrets`. `apps/cli/src/daemon.rs:859-869`
- **CLI-4** — `--demo` flag synthesizes a rate-limit error on first call but the fallback uses **real** API keys. `apps/cli/src/main.rs:866` + `apps/cli/src/agent.rs:88`
- **CLI-5** — Auth credentials stored as plaintext JSON at `~/.agiworkforce/auth.json` (0o600), not OS keyring. `apps/cli/src/auth.rs:189-197`

### Web

- **WEB-1** — `max_tokens` unbounded in `LLMCompletionRequestSchema`. Single request can drain a billing period. `apps/web/lib/validations/llm.ts:17`
- **WEB-2** — SSRF: `*_BASE_URL` env vars passed verbatim to `fetch()`. No hostname allowlist. `apps/web/lib/llm-providers/factory.ts:126-145`
- **WEB-3** — Streaming LLM proxy has no timeout on upstream `ReadableStream`. Stalled stream pins edge function until function-timeout. `apps/web/app/api/llm/completion/route.ts:388-514`

### Desktop

- **DESK-1** — `computer_use_execute_tool` allows `zoom` and `zoom_at_point` to bypass `require_confirmation`. Privacy leak via screen capture. `apps/desktop/src-tauri/src/sys/commands/computer_use.rs:346-428`
- **DESK-2** — Autopilot mode flag is callable from frontend with no auth gate, time-bound, or audit log. `apps/desktop/src-tauri/src/sys/commands/tool_confirmation.rs:521`
- **DESK-3** — `storage.rs::init_with_machine_key()` fallback runs whenever vault is locked, defeating the FIX-001 vault gate at the storage layer. `apps/desktop/src-tauri/src/sys/security/storage.rs:203-225`

### Chrome Extension

- **EXT-1** — `isValidSender()` accepts any `sender.tab`. Combined with all-origins manifest, any page (incl. XSS payload) can call privileged background commands. `apps/extension/src/background.js:27-39`
- **EXT-2** — Manifest matches `<all_urls>`, no user-controlled site allowlist. `apps/extension/manifest.json:30-39`
- **EXT-3** — `TYPE`/`CLICK`/`SET_LOCAL_STORAGE` commands have no same-tab restriction — cross-tab DOM mutation possible. `apps/extension/src/content.js:543-577`

## P1 — Real bugs (25)

### Desktop

- **DESK-4** — `messaging.rs::encrypt_messaging_credentials` returns plaintext when vault unconfigured. New users get plaintext Slack/WhatsApp tokens in DB. `apps/desktop/src-tauri/src/sys/commands/messaging.rs:23-38`
- **DESK-5** — `supabase_sync.rs` reads Vite-prefixed env vars from Rust process env. Always returns None in bundled app — silent cloud-sync failure. `apps/desktop/src-tauri/src/data/supabase_sync.rs:71-83`
- **DESK-6** — `migrations.rs` uses `format!("SAVEPOINT migration_v{}", version)` raw-string interpolation. Latent SQLi if version ever becomes attacker-influenced. `apps/desktop/src-tauri/src/data/db/migrations.rs:267-277`
- **DESK-7** — `autonomous.rs` budget check is `>` not `>=`. `max_session_cost=0.0` (default) silently halts every LLM call after first cost > 0. `apps/desktop/src-tauri/src/core/agent/autonomous.rs:178-201`
- **DESK-8** — `tool_confirmation` remembered choices + session approvals in-RAM only. Lost on crash, no shutdown drain. `apps/desktop/src-tauri/src/sys/commands/tool_confirmation.rs:38-53, 186-196`
- **DESK-9** — `computer_use_execute_tool` calls other `#[tauri::command]` fns directly in Rust, which is unsupported Tauri API misuse. `apps/desktop/src-tauri/src/sys/commands/computer_use.rs:378-382`
- **DESK-10** — `ManagedCloudProvider::default()` panics on TLS-builder failure (Alpine CI, minimal Windows installs). Wide blast radius via `#[derive(Default)]`. `apps/desktop/src-tauri/src/core/llm/providers/managed_cloud_provider.rs:57-62`
- **DESK-11** — `supabase.ts::tauriStorage.getItem` returns plaintext JWT from localStorage when vault write fails during migration. Token never migrates. `apps/desktop/src/lib/supabase.ts:93-106`
- **DESK-12** — `fs_search_files`/`fs_search_folders` root at `current_dir()` = `/` on macOS dock-launched app. Walks tens of thousands of system files. `apps/desktop/src-tauri/src/sys/filesystem/search.rs:17-33`

### Web

- **WEB-4** — `/api/stripe-webhook` body-read fragility: middleware mutates request before route handler reads raw body for HMAC. Latent signature-verification break. `apps/web/utils/supabase/proxy.ts:92-99`
- **WEB-5** — CSRF protection skipped for all Bearer-token requests. The web SPA uses Bearer auth — CSRF protection is inoperative for browser flow. `apps/web/lib/csrf.ts:207-213`
- **WEB-6** — `tools` and `tool_choice` are `z.unknown()` — forwarded verbatim to upstream LLM API. `apps/web/lib/validations/llm.ts:19`
- **WEB-7** — Stripe webhook handler returns `errorMessage` in 500 response body. Info disclosure (Supabase column names, SQL constraint names). `apps/web/app/api/stripe-webhook/route.ts:1687`
- **WEB-8** — `/api/sync-subscription` falls back to Stripe customer search by email when no `stripe_customer_id`. IDOR if user changed email. `apps/web/app/api/sync-subscription/route.ts:64`
- **WEB-9** — `/auth/callback` does not validate OAuth `state` parameter. PKCE partially mitigates but state nonce is required by RFC 6749. `apps/web/app/auth/callback/route.ts:6-56`
- **WEB-10** — `error-handler.ts::handleError` returns `error.message` + `error.details` verbatim for `AppError`. Supabase error details leak via wrap-and-rethrow. `apps/web/lib/error-handler.ts:24-35`
- **WEB-11** — Web CSP has `style-src 'unsafe-inline'` (acknowledged in code comment, not in FIX-005 scope). `apps/web/proxy.ts:20`
- **WEB-12** — Supabase `setAll` strips cookie options on request-side mutation. Possible session-cookie security flag drop. `apps/web/utils/supabase/proxy.ts:30-38`

### Mobile

- **MOB-1** — Push token sent to backend via `getAuthHeaders()` race window — can fire before MMKV encryption + Supabase session resolve. Unauth'd push-token registration. `apps/mobile/services/notifications.ts:174-183`
- **MOB-2** — Deep-link handler validates code regex but not URL scheme. Malicious app on Android can register `agiworkforce://` and inject pairing codes. `apps/mobile/app/_layout.tsx:196-213`
- **MOB-3** — `secureStorage.setItem` swallows SecureStore errors. Lost auth-token writes cause silent sign-out on next cold start. `apps/mobile/lib/secureStorage.ts:27-36`
- **MOB-4** — `secureGet` returns null in `Before-First-Unlock` keychain state. Push-notif cold-start before unlock destroys session silently. `apps/mobile/services/supabase.ts:28-34`

### VS Code Extension

- **EXTV-1** — `autoApplyFixes` setting respected from workspace config without `vscode.workspace.isTrusted` check. Untrusted repo can opt itself in to LLM auto-apply. `apps/extension-vscode/src/extension.ts:1063`
- **EXTV-2** — Agent-mode WebviewPanel has `enableScripts: true` with no CSP set in HTML. Sidebar provider correctly uses nonce-based CSP — agent panel doesn't. `apps/extension-vscode/src/providers/agentModeProvider.ts:131-144`
- **EXTV-3** — `terminal.sendText(cmd)` invoked without workspace-trust gate from `agi-workforce.runCommand` and `agi.test.run`. `apps/extension-vscode/src/providers/terminalProvider.ts` + `apps/extension-vscode/src/extension.ts:903-929`

### API Gateway / Signaling

- **GW-1** — `mobile.ts` router applies rate-limiter at line 31 but `authenticateToken` only at line 44. Pattern inconsistent with `desktop.ts`. Future route insertion would silently bypass auth. `services/api-gateway/src/routes/mobile.ts:31`
- **GW-2** — Latent token leak: LLM proxy correctly builds new headers but a future refactor copying `req.headers` would forward the user's `Authorization` to Anthropic/OpenAI. No CI lint blocks this. `services/api-gateway/src/routes/llm.ts:200`
- **SIG-1** — Origin validation allows no-Origin connections (for desktop/mobile native clients) without alternate auth. SSRF / mobile-misconfig can bypass entirely. `services/signaling-server/src/index.ts:715-722`
- **SIG-2** — `control` signal kind forwards `message.payload` (unvalidated) to peer instead of the Zod-parsed value. `services/signaling-server/src/index.ts:1285-1329`
