# Wave 3 launch playbook

> Mobile (App Store + Play) + Chrome ext (Web Store) + VS Code ext (Marketplace) + Hobby tier launch.
>
> Last updated: 2026-05-04 — Wave 3 kickoff.

This playbook is the operator's checklist for shipping Wave 3. Each item is tagged:

- **🟢 codeable** — done or doable in a code session, no external accounts/keys/dashboards needed
- **🟡 config** — needs values in Stripe / EAS / env / dashboard, but no new code
- **🔴 human** — requires a developer account, signature/cert, screenshot, or marketplace approval

The big shape of Wave 3 is **distribution, not engineering**. Most code is already shipped (S1–S13 closed all the LLM/agent infra; Hobby tier is wired). What's left is signing, listings, and dashboard config.

## Operator launch order

If you're shipping all four in one push, the dependency-aware order is:

1. **Hobby tier** first (no review queue; flips on instantly once Stripe is configured).
2. **VS Code extension** next (no review queue; instant publish via `vsce`).
3. **Chrome extension** (2–7 day review).
4. **Mobile (Play first, App Store last)** — Play tends to clear in 1–7 days; Apple is the long pole.

This gives Hobby revenue + VS Code distribution before the long-tail mobile review.
