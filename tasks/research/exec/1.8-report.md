## Per-package wiring

### `@agiworkforce/mcp` — MCP transport-discriminated client

**Files added/modified**:

- `apps/desktop/src/services/mcp.ts` (new) — TS-side wrapper over the
  shared package. Exposes `connectDesktopMcpServer`,
  `buildDesktopMcpCatalog`, `probeMcpServer`. Used in cloud-mode desktop
  (no Tauri) and direct-from-renderer connections; the existing
  Tauri-Rust MCP path under `apps/desktop/src-tauri/` remains untouched
  (Rust-only stack stays Rust-owned).
- `apps/web/app/api/mcp/route.ts` (new) — `POST /api/mcp` connect-and-list
  endpoint. SSRF-defensive: rejects stdio transports outright, validates
  HTTPS URLs against a private-network blocklist that mirrors the
  api-gateway's `mcpConfig.ts` policy. CSRF-protected; auth-required;
  rate-limited via the canonical `chat-conversation` policy. Uses
  `runtime = 'nodejs'` (the MCP SDK needs Node APIs).
- `apps/mobile/services/mcp.ts` (new) — `import type` only from
  `@agiworkforce/mcp` (Metro tree-shakes cleanly, no Node-only code
  reaches the bundle). Wraps `secureFetch` against
  `/api/mcp/servers/...` on the api-gateway. Supports `listMcpServers`,
  `listMcpTools`, `callMcpTool`. Auth via Supabase Bearer token.
- `services/api-gateway/src/mcp/sharedClient.ts` (new) — shared-package
  proxy for the api-gateway. `getSharedMcpCatalog()` builds a flat
  catalog from the `mcpConfig.ts` allowlist, with 60-second cache and
  in-flight-coalesced concurrent calls. `callSharedMcpTool()` lazily
  ensures the server handle is live and reuses it. `closeAllSharedMcpHandles()`
  for graceful shutdown.
- `services/api-gateway/src/mcp/mcpProxy.ts` (modified) — added
  deprecation header + first-use deprecation warning logged to pino. The
  legacy hand-rolled JSON-RPC implementation remains for the existing
  `/api/mcp/...` routes during the migration window; new code consumes
  `sharedClient.ts`.

**Edge cases covered**:

- **8 transport variants exposed**: today's shared package supports
  stdio + SSE + streamable-HTTP. The remaining 5 (`ws`, `ws-ide`,
  `sse-ide`, `sdk`, `claudeai-proxy`) require additional MCP-SDK plumbing
  and are tracked as a follow-up — the gap-matrix entry §B.3 lists each
  with its reference LOC. The `resolveMcpTransport` discriminator is
  designed to fall through to "transport not supported" rather than
  crashing when one is requested.
- **Backward-compat**: `mcpProxy.ts`'s `getMcpProxy()` keeps working;
  emits a one-shot deprecation warning so operators see the migration
  signal without spamming logs.

### `@agiworkforce/skills` — layered skill loader

**Files added/modified**:

- `apps/desktop/src/lib/skillLoader.ts` (modified) — added
  `loadFilesystemSkills()` and `formatFilesystemSkills()` async helpers
  that lazy-load `@agiworkforce/skills` via dynamic import. Guarded with
  a runtime-environment check (`typeof window !== 'undefined' && typeof
process === 'undefined'` → skip) so calling from a renderer context
  short-circuits cleanly. The existing Vite `import.meta.glob`-based
  bundled-employee loader is preserved with an `@deprecated` JSDoc
  marker pending the appStateStore catalog slice landing.
- `apps/web/features/chat/components/SkillsMenu.tsx` (new) —
  consumer-facing component. Pulls metadata from `/api/skills`,
  body-on-expand from `/api/skills/[name]`. Implements the progressive-
  disclosure UX with a "Loading skill body…" placeholder during the
  body fetch. Type-imports `Skill` from `@agiworkforce/skills` so
  `SkillSummary` stays in sync with the canonical wire shape.
- `apps/web/app/api/skills/route.ts` (new) — `GET /api/skills` returns
  metadata only; never includes body content. Layers configured via
  `SKILLS_LAYERS` env var (JSON `[{rootDir, source}]`). 60-second cache.
- `apps/web/app/api/skills/[name]/route.ts` (new) — body endpoint, lazy
  step. 200-char name length cap.
- `apps/mobile/services/skills.ts` (new) — `import type { Skill }` from
  the shared package; runtime is plain `secureFetch`. `listSkills` +
  `getSkillBody` mirror the web component's progressive-disclosure
  pattern. Bundle delta: <1 KB minified+gzipped (types only).
- `services/api-gateway/src/services/skillsCatalog.ts` (new) —
  `getSkillCatalog()` returns metadata only; `getSkillBody()` is the
  lazy step. `setSkillsLayers()` is called once at gateway startup;
  60-second cache; `invalidateSkillsCatalog()` for tests.

**Edge cases covered**:

- **`paths` gitignore-glob conditional activation**: the shared package
  doesn't yet ship the `paths` activation logic (gap matrix C.3 marks
  it as the marquee missing feature). When it lands, consumers don't
  need to change — `loadSkillsFromLayers()` will then return skills
  whose `metadata.paths` populates and the consumer routes will get
  filtering by file-touch events for free.
- **Progressive disclosure**: list endpoint never returns body content;
  bodies fetched per skill on demand. Consumer UIs (web `SkillsMenu`,
  mobile `getSkillBody`) implement the "Loading skill body…" feedback.
- **Mobile bundle size**: `import type` only — Metro tree-shakes
  cleanly. Verified: zero transitive resolution of
  `@agiworkforce/skills`'s `node:fs/promises` import reaches the Metro
  output. Effective delta: ~0 KB.

### `@agiworkforce/apply-patch` — OpenClaw-format patch applicator

**Files added/modified**:

- `services/api-gateway/src/tools/file_edit.ts` (new) — server-side
  `applyFileEdit({patch, workspaceRoot, signal})` wrapper. Adds a
  defense-in-depth blocklist of system-sensitive workspace roots
  (`$HOME`, `/etc`, `/var`, `/root`, `/usr`); the package's
  `WorkspaceEscapeError` already enforces lexical + realpath canonical
  path-escape rejection. Returns a typed result union with `code:
'workspace_escape' | 'parse_error' | 'apply_error' | 'invalid_input'`.
- `services/api-gateway/src/tools/__tests__/file_edit.test.ts` (new) —
  4 integration tests covering happy path (Add patch in tmpdir),
  workspace-escape rejection (`../escape.txt`), system-blocklist root
  rejection (`/etc`), and invalid input. **All 4 passing.**
- `apps/desktop/src/services/applyPatch.ts` (new) — renderer-side
  wrapper exposing `applyDesktopPatch(patchText, {cwd, fs, signal})`.
  Re-exports `WorkspaceEscapeError` and `FSBridge` so callers can pass
  a Tauri-FS-backed bridge for renderer-context patch application.

**Edge cases covered**:

- **Cross-platform path semantics**: the package uses `node:path`
  which auto-selects `/` on POSIX and `\` on Windows. `path.resolve()`
  normalizes mixed separators (e.g., a model-emitted `src/foo` works
  on Windows). The `realpath` canonicalization layer additionally
  defeats trojan-symlink attacks (relevant on POSIX; on Windows,
  `node:fs` honors NTFS reparse points). UNC-path handling is a known
  gap (gap-matrix D.3) — not yet in scope.
- **Backward-compat**: no existing apply-patch consumers to break; the
  wrapper is greenfield. Future migration of any in-tree
  `multi_edit`-style code would route through `applyFileEdit()`.

## Edge-case decisions worth flagging to team-lead

1. **CLI Rust skill stack untouched** as instructed. The `apps/cli/src/
skills.rs` Rust loader continues to own CLI skills; `@agiworkforce/
skills` is consumed by TS surfaces only.
2. **Mobile mcp/skills are HTTP proxies** rather than direct package
   consumers. React Native cannot run `@modelcontextprotocol/sdk`
   (no `child_process`, no `node:fs`) nor read disk for skill files.
   Mobile shares the canonical wire types via `import type` and routes
   through the api-gateway — the simplest pattern that keeps the
   wire shape strictly typed without bloating the JS bundle.
3. **`zoom` action returns an `unsupported` step** in the extension
   bridge. The model still receives a structured response (it can
   re-plan with a different action), and we don't crash the action
   loop. Implementing `zoom` would require the extension's `tabs`
   permission and an extension-side dispatcher; not in scope for this
   task.
4. **Web `/api/mcp` rejects stdio transports outright**. Spawning child
   processes inside a Next.js route handler is the wrong layer; the
   api-gateway already has a vetted allowlist for stdio servers
   (`mcpConfig.ts`). The web route is for HTTP-family transports only.
5. **Backward-compat shim retention**: rather than nuking the legacy
   `mcpProxy.ts`, I added a deprecation log path. Migration of the
   `/api/mcp/...` routes to `sharedClient.ts` is a separate ticket — a
   straight find-and-replace risks routing-table regressions during
   ship-blocker remediation, so the cleaner approach is to migrate
   per-route after the foundation merge.
