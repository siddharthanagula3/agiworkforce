## Files Created

### `packages/runtime/src/state/createStore.ts` (line 1–62)

34 substantive LOC (confirmed via `grep -v comment | wc -l`). Ports `state/store.ts:10-34` from the Anthropic reference verbatim:

- `Object.is` short-circuit at line 52 prevents render storms on per-keystroke mutations (e.g. streaming flag toggling at 60 FPS would produce 0 re-renders if the value doesn't change).
- `onChange` fires **before** listeners (line 53): by the time React re-renders, all side effects in `onChangeAppState` are settled. This is critical for React 19's concurrent mode where effects fire after paint — if onChange fired after listeners, the first render cycle would see stale side-effect state.
- `subscribe` returns `() => void` (line 55–57), which is exactly the signature `useSyncExternalStore` expects for its first argument. Components using this store with `useSyncExternalStore` will never use `useState` for subscription — they get correct concurrent-render semantics for free.

### `packages/runtime/src/state/onChangeAppState.ts` (line 1–265)

Single choke-point with 4 fan-out channels:

**Channel 1 — API client cache invalidation** (`registerApiCacheInvalidator`, line 75–94): Fires when `auth.userId`, `auth.accessToken`, `auth.planTier`, or `chat.activeModelId` changes. API adapters (e.g. the OpenAI adapter) register a callback that busts their in-flight cache. This eliminates the "stale JWT on concurrent request" bug class where 8+ mutation paths previously each had to manually call `clearCache()`.

**Channel 2 — Telemetry** (`registerTelemetryHandler`, line 111–153): Emits an `AppStateTelemetryEvent` with `changedFields: string[]` and `ts: number` on every change that touches a tracked field. The field list covers: `auth.{userId,planTier,isAuthenticated}`, `chat.{activeModelId,activeConversationId,isStreaming}`, `settings.{theme,language}`, `subscriptions.planTier`, `mcp.connectedCount`. Events with no changed tracked fields are silently skipped (line 142).

**Channel 3 — Settings persistence** (`registerPersistenceHandler`, line 162–176): Fires when `prev.settings !== next.settings` (reference comparison — no deep diff). Persistence backends register a handler: desktop writes to `~/.agiworkforce/state.json`, web to localStorage, mobile to MMKV (registered in the mobile workspace). The full persistence implementation is deferred to follow-on; the wiring point is here.

**Channel 4 — Model-switch broadcast** (`registerModelSwitchListener`, line 192–213): Fires when `chat.activeModelId` changes. Emits `{prevModelId, nextModelId, source: 'app_state'}`. Cross-surface consumers (Chrome extension bridge on port 8787, VS Code extension) register here to receive model-switch notifications without polling. **Model IDs are never hardcoded** — `prevModelId`/`nextModelId` are whatever the store resolved from `models.json` at runtime.

**Circular re-entrancy guard** (line 220–228): Each call carries a `depth` counter (default 0). If `depth > MAX_FANOUT_DEPTH` (2), the call emits a `CircularFanOutError` and returns immediately, preventing Store A → onChange → Store B → onChange → Store A infinite loops. The depth is passed explicitly by the caller (not tracked in module state), so it's testable and reset between distinct call chains.

**Fan-out failure isolation** (lines 231–248): Each channel wrapped in `try/catch`. One failing channel emits a `FanOutError` via `onFanOutError` and continues to the next channel. Consumers can register error listeners for observability without crashing the app.

## Architectural Decisions

1. **Bridge over rewrite**: The 12 stores are bridged (not rewritten) to avoid breaking 1622 existing tests and all consumer components. The bridge is a thin `subscribe` adapter with `Object.is` guards — it adds <1ms latency per mutation.

2. **Depth counter vs. module-level flag for circularity**: A module-level `isProcessing` flag would prevent re-entrancy globally but would also block legitimate concurrent calls from different chains. The depth counter is passed per-call, so Store A → Store B (depth 1) and a concurrent Store C → Store D (depth 0) are independent.

3. **`onChange` before listeners**: Adopted directly from the Anthropic reference. This means by the time React re-renders, all side effects (cache invalidation, telemetry, persistence, model broadcast) are already done. The alternative (onChange after listeners) would cause a second render cycle for side effects that modify derived state.

4. **6-domain consolidation**: Rather than a flat 75-field store (Anthropic reference), we use 6 typed sub-objects (`auth`, `chat`, `settings`, `subscriptions`, `mcp`, `memory`). This lets `useSyncExternalStore` selectors be written as `(s) => s.auth.planTier` without destructuring the entire store, and lets Object.is short-circuit at the domain level in onChangeAppState.

5. **No hardcoded model IDs**: `chat.activeModelId` is `string | null`, initialized to `null`. The bridge from `modelStore.ts` (bridge 5 in `stateBridge.ts`) forwards whatever `selectedModel` the model store resolved from `models.json`. The locked rule is enforced at both the type level (no literal union) and the initial state (`null`, not a string).
