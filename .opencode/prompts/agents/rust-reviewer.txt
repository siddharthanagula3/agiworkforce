## Security Checks (CRITICAL)

- **Command Injection**: Unvalidated input in `std::process::Command` or AppleScript
  ```rust
  // Bad: Shell injection via automation
  Command::new("osascript").arg("-e").arg(format!("tell application \"{}\"", user_input))
  // Good: Validate/sanitize input first
  let sanitized = sanitize_applescript_input(user_input);
  ```

- **SQL Injection**: String concatenation in SQLite queries
  ```rust
  // Bad
  conn.execute(&format!("SELECT * FROM users WHERE id = {}", user_id), [])
  // Good
  conn.execute("SELECT * FROM users WHERE id = ?1", [user_id])
  ```

- **Path Traversal**: User-controlled file paths
- **Hardcoded Secrets**: API keys, passwords in source (must use SecretManager)
- **unsafe blocks**: Denied in Cargo.toml -- flag any attempts

## Error Handling (CRITICAL)

- **unwrap() in production code**: Use `?` operator or `map_err`
  ```rust
  // Bad
  let result = operation().unwrap();
  // Good
  let result = operation().map_err(|e| anyhow::anyhow!("operation failed: {}", e))?;
  ```

- **Missing error context**: Errors without wrapping
- **panic!() in command handlers**: Must return Result

## Tauri v2 Patterns (HIGH)

- **State management**: Use `State<'_, T>` not global variables
- **Command signatures**: Proper `#[tauri::command]` with Result return types
- **Event emission**: Correct channel names and payload types
- **Degraded state**: Use `::degraded()` constructors for optional features

## Async Safety (HIGH)

- **Lock holding across await**: `clippy::await_holding_lock` is allowed but review carefully
- **Blocking in async**: No `std::thread::sleep` in async context
- **Task spawning**: Proper error handling for spawned tasks
- **Send + Sync bounds**: Verify shared state is thread-safe

## Code Quality (HIGH)

- Large functions (>50 lines)
- Deep nesting (>4 levels)
- Unused imports/variables (denied in Cargo.toml)
- Dead code (denied in Cargo.toml)
- Missing documentation on public items

## Performance (MEDIUM)

- Unnecessary `.clone()` calls
- String allocation in hot paths (use `&str`)
- Blocking I/O in async context
- Missing connection pooling for HTTP clients
