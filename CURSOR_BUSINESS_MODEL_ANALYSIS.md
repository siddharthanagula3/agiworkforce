## How They Afford Generous Pricing

### The Math Behind Cursor's Strategy

**Cursor's Current Pricing (2025):**

```
Hobby (Free):    Unlimited Auto mode + 50 messages/month with premium models
Pro ($20/month):  Unlimited Auto + $20 in credits for premium models
Ultra ($200/month): Unlimited Auto + ~$200 in credits for premium models
```

**Their Cost Structure:**

- Claude Opus (Anthropic): ~$0.08-0.10 per request
- GPT-4o (OpenAI): ~$0.06-0.08 per request
- Gemini Pro (Google): ~$0.01-0.02 per request
- Infrastructure/Hosting: ~$2-5 per active user per month
- Support: ~$1-3 per paying user

**The Old Problem (Pre-June 2025):**

```
User on Pro plan ($20/month) making ~300 Claude Opus requests
Cursor's cost: 300 × $0.10 = $30 in model credits
+ Infrastructure + Support = $35-40 per user
Revenue: $20
Loss per user: -$15-20/month per heavy user
```

**Why this was unsustainable:** They couldn't subsidize heavy users with VC cash forever.

## The Hidden Cost of Cursor's Strategy

### What Investors Don't Talk About

**Churn Risk:**

```
If Cursor raises prices to $50/month, free users don't churn (they're free)
But Pro users might switch to GitHub Copilot or other tools
Risk: They build a customer base that's not loyal
```

**Competitive Risk:**

```
GitHub Copilot is improving rapidly
Microsoft is investing heavily
OpenAI could launch their own editor
Risk: Cursor loses differentiation
```

**Acquisition Risk:**

```
All $3.3B in funding is not free
VC investors expect 10-30x returns
If acquired for $15B (good deal), LPs make 4-5x
But most VC-backed companies don't achieve this
Risk: Investors force sale at wrong price, or company fails
```

**Profitability Risk:**

```
Raising prices to become profitable could cause mass churn
Network effects not strong enough yet
Risk: The moat disappears as soon as they raise prices
```
