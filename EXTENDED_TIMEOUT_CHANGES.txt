===== EXTENDED TIMEOUT SUPPORT: SUMMARY OF CHANGES =====

PROJECT: AGI Workforce Extended Task Timeout Support
FEATURE: Enable 30+ hour autonomous AGI task execution
DATE: 2026-02-01

===== OVERVIEW =====

Extended AGI Workforce to support task timeouts from 1 minute to 72 hours,
with persistent task storage, checkpointing, timeout warnings, and graceful
degradation. Previously limited to 5 minutes (300 seconds).

===== KEY FEATURES ADDED =====

1. Configurable Task Timeouts
   - Min: 1 minute, Max: 72 hours, Default: 24 hours
   - Per-user setting in Settings → Execution Preferences
   - Per-task override support via Goal constraints
   - Setting validation and clamping

2. Real-time Timeout Tracking
   - Elapsed time calculation (excluding paused periods)
   - Remaining time countdown
   - Progress percentage (0-100)
   - Automatic warning emissions at 1hr, 30min, 5min

3. Graceful Timeout Handling
   - User warning dialog with action choices
   - Extend timeout by custom minutes
   - Pause and resume later
   - Continue or abort task
   - Warnings reset after extension

4. Persistent Task Storage
   - SQLite backend with WAL mode
   - Automatic task state serialization
   - Checkpoint creation at configurable intervals
   - Status tracking: queued, running, paused, completed, failed, cancelled

5. Progress Checkpointing
   - Execution context snapshots
   - Tool results preservation
   - Step-level granularity
   - Automatic cleanup of old checkpoints

6. App Restart Recovery
   - Auto-resume paused/queued tasks
   - Restore execution context from latest checkpoint
   - Preserve task priority and metadata

===== ARCHITECTURE CHANGES =====

Previous (5-minute limit):
  BackgroundAgent → Timeout (hardcoded 300s) → Task expires → Fail

New (Extended timeout):
  SettingsStore (TimeoutConfig)
         ↓
  AGICore/BackgroundAgent
         ↓
  TimeoutTracker (real-time tracking)
         ↓
  TimeoutWarning (1hr, 30min, 5min)
         ↓
  User Choice Dialog
         ├─ Extend timeout → Add time
         ├─ Continue → Keep running
         ├─ Pause Later → Save and stop
         └─ Abort → Cancel task
         ↓
  TaskStorage (SQLite persistence)
         ├─ persistent_tasks table
         ├─ task_checkpoints table
         └─ Auto-resume on app restart

===== DATABASE SCHEMA ADDED =====

Table: persistent_tasks
  - id (TEXT PRIMARY KEY)
  - description, status, priority
  - current_step, total_steps
  - timeout_secs, elapsed_secs
  - created_at, started_at, completed_at (RFC3339 timestamps)
  - context_json, progress_json (serialized state)
  - notes (optional user notes)

Table: task_checkpoints
  - id, task_id (FOREIGN KEY)
  - step_number, context_json, tool_results_json
  - created_at, metadata_json
  - Indexes: task_id, step_number
